OpenCV

OpenCV is installed on the server. You need to use "-I/usr/local/include/opencv -L/usr/local/lib -lcxcore -lcv -lhighgui -lcvaux -lml" in compiling the code. Alternatively, you can also use: "g++ -o my_example my_example.cpp `pkg-config opencv --cflags --libs`".

VNC

VNC is used to remotely log in to the server. To set up, firt use putty to login to the server. Run: 

 
% vnc4server
You should be able to see something like the following: 
You will require a password to access your desktops.

Password:
Verify:
Found /usr/share/tightvnc-java for http connections.

New 'X' desktop is ir:1

Creating default startup script /home/yazhang/.vnc/xstartup
Starting applications specified in /home/yazhang/.vnc/xstartup
Log file is /home/yazhang/.vnc/ir:1.log
Copy the xstartup file from my directory to yours. Then run: 
% vnc4server -kill :1.
% vnc4server

On your windows PC, download TightVNC: download link. After installation, open TightVNC viewer, type ir.sjtu.edu.cn:N (N is the number you see after "ir:" when you start the vncserver) as the TightVNC Server, then click connect. You should be able see a graphic user interface for it after it.

Solution for keyboard problem

Open dconf-editor, and search for "keybind" (since I was at the physical machine I had no problem typing the "d"). This enabled me to find org->gnome->desktop->wm->keybindings. In the right pane for org->gnome->desktop->wm->keybindings, I then looked for all settings that included and deleted those settings. Then killed VNC server, started it again,
